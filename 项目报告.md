## 一、选题动机

在信息过载时代，学生和职场人每天需要快速处理大量网页长文、论文摘要或会议记录。手动阅读全文耗时且容易遗漏重点。以自身为例，每周需阅读多篇万字级资料，核心信息提取效率较低。

该工具通过 AI 将"线性阅读"转化为"结构化速读"，自动按"核心结论→关键要点→行动启发"三段式输出，帮助用户在 3 秒内获取文本精髓，解决信息焦虑问题，具备明确的实用价值。

## 二、技术方案

本系统采用**前后端一体化 + AI 代理架构**，确保功能闭环稳定运行。

### 系统架构

```
用户浏览器 ──POST/文本──→ FastAPI 后端 ──API 请求──→ 硅基流动 LLM
     ↑                        │                            │
     └──────── HTML 响应 ─────┘                            │
                        └────────── 结构化 JSON ────────────┘
```

### 模块说明

| 模块 | 技术选型 | 功能 |
|------|----------|------|
| 前端 | 原生 HTML + CSS + JavaScript，marked.js 渲染 | 文本输入、请求发送、Markdown 结果展示 |
| 后端 | Python FastAPI | API 代理转发、CORS 跨域处理、密钥保护 |
| AI 引擎 | 硅基流动 SiliconFlow（OpenAI 兼容接口）/ Qwen-2.5-72B-Instruct | 文本结构化总结生成 |
| 部署 | Hugging Face Spaces（Docker） | 云端上线，公网可访问 |

### 关键设计决策

1. **后端代理而非前端直连**：防止 API Key 暴露在浏览器端，同时解决跨域问题
2. **文本截断保护**：限制单次输入 2000 字符，防止超长文本导致 API 超时或费用失控
3. **Prompt 工程引导**：通过结构化指令控制模型输出格式，确保结果一致性

## 三、核心功能演示

### 3.1 首页界面

页面提供简洁的文本输入框与"一键总结"按钮，用户粘贴任意长文本即可触发处理。界面采用白蓝配色，聚焦核心交互流。

### 3.2 处理中状态

点击按钮后，UI 显示加载动画"🤖 AI 正在分析提炼中"，按钮变为禁用状态并显示"⏳ AI 分析中"，防止重复提交。后台异步调用大模型，不阻塞页面。

### 3.3 结构化输出

AI 返回结果后，前端自动解析 Markdown，按以下格式展示：

- **📌 核心结论**：文本的核心论点摘要
- **🔑 关键要点**：分点列出重要信息
- **💡 行动/启发**：基于内容的 actionable 建议

响应时间稳定在 2-4 秒。功能完整跑通"感知（用户输入）→ 推理（API 分析）→ 行动（界面反馈）"闭环。

## 四、开发过程与反思

### 4.1 遇到的困难与解决

| 困难 | 解决方案 |
|------|----------|
| 前端直连 API 遭遇 CORS 跨域拦截 | 改为 FastAPI 后端代理转发 |
| Python 环境变量未配置导致密钥泄露风险 | 使用 .env 文件 + 环境变量管理密钥 |
| Hugging Face Spaces 部署时 static 目录不存在 | 将 HTML 内联嵌入 main.py，消除目录依赖 |

### 4.2 AI 协作效率

代码整体生成率约 80%。AI 在生成基础框架、API 调用、前端渲染方面高效可靠，但在以下环节需要人工干预：

- **架构决策**：选择后端代理还是前端直连
- **环境适配**：不同平台的部署配置差异
- **异常处理**：网络超时、空响应的健壮性

### 4.3 不足与改进方向

目前版本仅支持纯文本输入，尚未集成网页爬虫自动提取正文（如 trafilatura）。未来可加入"输入 URL 自动解析"功能，实现全自动网页总结。整体遵循 MVP 原则，优先保证核心链路畅通。

## 五、AI 协作记录

| 环节 | AI 贡献（Prompt 示例） | 人工判断与修正 |
|------|----------------------|----------------|
| 架构设计 | "用 FastAPI + 纯前端写一个文章总结工具，要最简单能跑" | 决定将 HTML 嵌入后端，消除文件目录依赖 |
| 后端代码 | "写一个调用 OpenAI 接口的 POST 接口" | 替换 URL 为硅基流动兼容端点，添加 2000 字符截断 |
| 前端渲染 | "用原生 JS 调用后端 API，用 Markdown 展示结果" | 引入 marked.js CDN，优化加载状态和错误处理 |
| 部署配置 | "Docker 部署 Python FastAPI 到 Hugging Face Spaces" | 手动添加环境变量 Secret，调试端口映射 |

**总结**：AI 负责"砖块"生成，人工负责"图纸"设计与"质检"。人机协同的关键不在于用 AI 替换程序员，而在于用 AI 放大人类的定义问题和决策能力。
